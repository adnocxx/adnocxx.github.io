接口

在ros中，我们用的消息类型都是在功能包的msg文件夹下，通常这个包叫做接口包。

查看接口包里面所有的接口

例如ros2 interface package sensor_msgs

自定义接口

常用接口命令

创建接口其实就是创建一个接口格式的文件，放在功能包的msg文件下
其中，有三种接口类型：

  • srv：服务接口
  • msg：消息接口
  • action：动作接口

eg:ros2 interface package std_msgs

查看某一个接口详细的内容

ros2 interface show std_msgs/msg/String

输出某一个接口所有的属性

ros2 interface proto sensor_msgs/msg/Image

创建接口流程

比如ros中的std_msgs/msg/String，这个接口定义了一个字符串,他是依赖基本的数据类型string，并且string data，所以在
输出数据时必须用这个接口，内容是.data

from std_msgs.msg import String
msg = String()
msg.data = '第%d回：潋滟湖 %d 次偶遇胡艳娘' % (self.i,self.i)
# msg 是 std_msgs.msg.String() 的对象
# msg.data data是string类型的对象，其定义是string data

服务端与客户端

话题与服务的区别

  • 话题是发布订阅模型。主要是单向传输数据，只能由发布者发布，接收者接收（同一话题，发布者接收者都可以有多个）
  • 服务是客户端服务端（请求响应）模型。由客户端发送请求，服务端处理请求，然后返回处理结果（同一服务，客户端可以由多个，服务端只能有一个）

一个节点中包含服务端，定义的时候有回调函数和接口，可以在客户端发出请求后，执行回调函数，在回调函数中处理信息并返回结果，以及打印信息以及执行结果反馈。
